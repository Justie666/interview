## Как добавить новый вопрос

### 1. Определи раздел

| Тема вопроса                                           | Файл            |
| ------------------------------------------------------ | --------------- |
| JavaScript (Event Loop, замыкания, прототипы, типы...) | `js.md`         |
| React (хуки, рендеринг, контекст, Suspense...)         | `react.md`      |
| TypeScript (типы, утилиты, дженерики...)               | `typescript.md` |
| Браузер, HTTP, сеть, безопасность                      | `browser.md`    |
| Кодинг-задачи (quiz, реализации, найди баг)            | `tasks.md`      |

## Как добавить задачу в tasks.md

Задачи в `tasks.md` делятся на два типа:

**Quiz** — угадай вывод / объясни поведение:

```md
### N. Название задачи

\`\`\`js
// код
\`\`\`

<details>
<summary>Ответ</summary>

Объяснение + вывод.

</details>

---
```

**Реализация** — напиши с нуля:

```md
### N. Название задачи

Условие задачи одним абзацем.

\`\`\`js
// пример использования
\`\`\`

<details>
<summary>Решение</summary>

\`\`\`js
// полное решение
\`\`\`

**Ключевые моменты:** список важных нюансов.

</details>

---
```

### 2. Формат вопроса

Добавляй в конец нужного файла (перед последним `---`) в таком формате:

```md
## N. Заголовок вопроса в виде вопроса?

Короткое определение **ключевого понятия** жирным.

**Подзаголовок (если нужен):**

- Пункт 1
- Пункт 2

\`\`\`js
// пример кода
\`\`\`

Вывод или пояснение одной-двумя фразами.

---
```

### 3. Правила оформления

- Заголовок `##` — всегда в форме вопроса
- Номер вопроса — продолжение нумерации в файле (текущий максимум смотри в конце файла)
- Код — всегда в фenced-блоке с языком: ` ```js `, ` ```ts `, ` ```jsx `
- Таблицы — для сравнений (var/let/const, useEffect/useLayoutEffect и т.д.)
- Завершай вопрос горизонтальной линией `---`

### 4. Пример команды пользователю

Просто скажи: **"добавь вопрос про [тему]"** — и я найду нужный файл, определю номер и добавлю в правильном формате.

## Запуск

```bash
pnpm docs:dev      # локальный сервер
pnpm docs:build    # сборка
pnpm docs:preview  # превью сборки
```

Деплой — автоматически при пуше в `main` через GitHub Actions.
